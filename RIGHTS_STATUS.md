# Rights Status

## 現在の方針

このリポジトリは現時点で public preview として公開することを想定しているが、
ライセンス方針はまだ最終決定していない。

そのため、いま大事にしたいのは次の2点である。

1. このプロジェクトの発想、思想、実装の来歴を明確に残すこと
2. 将来の公開ライセンスや事業化の選択肢を早い段階で固定しすぎないこと

## いまの理解

- GitHub で公開すること自体は、発想と実装の存在を示す公開記録になる
- ただし、どのような条件で再利用を許すかは別途決める必要がある
- OSS ライセンスを付けない限り、原則として著作権は作者に留保される

## 将来の選択肢

### 1. しばらくライセンス未定のまま公開する

- public preview として見せる
- 実装と思想の発表を優先する
- 再利用条件は後で決める

### 2. コードを copyleft で公開する

- 候補: `AGPL-3.0`
- ネットワーク越しの利用も含めて改変コードの公開を求めやすい
- ただし守れるのは主にコードであり、アイデアそのものではない

### 3. コードと文書でライセンスを分ける

- コード: `AGPL-3.0` など
- 文書や思想説明: `CC BY-SA` など
- 何を開くか、どこまで共有を求めるかを分離できる

## 当面のおすすめ

まずは public preview / v0.1 として見せながら、

- README
- VISION
- PHASE1_MILESTONE
- RELEASE_v0.1

で思想と到達点を強く残す。

そのうえで、

- 本当に開いていくならライセンスを付ける
- まだ迷うなら一旦ライセンス未定のまま進める

という順番を取る。
