# VISION

## Core Message

### あなたの、絶対に残したい1秒はなんですか？

このプロジェクトの出発点は、データではなく問いにある。

人が本当に残したいのは、長い記録ではなく、人生の中のごく短い、しかし二度と取り戻せない一瞬かもしれない。
産声、声、しぐさ、手紙のような言葉。
この問いは、ユーザーに「何を保存するか」ではなく、「何を未来へ託したいか」を考えてもらうための入口である。

### ずっと、のこせるデジタル

このプロジェクトが目指しているのは、便利なメディア共有ではない。

目指しているのは、時間が経っても、サービスが終わっても、アプリがなくなっても、なお残るデジタルである。
クラウドに預けるのではなく、自分の手元に残ること。
一時的に見られることではなく、何年もあとで再び取り出せること。

「ずっと、のこせるデジタル」は、このプロジェクトのサービス価値を最も短く言い表す言葉である。

### 紙が、永遠のサーバーになる

このプロジェクトの技術思想はここに集約される。

QRコードの中にデータそのものを埋め込み、必要ならデコーダの入口や仕様まで紙に残す。
そうすると、紙は単なる印刷物ではなく、保存媒体そのものになる。

サーバーに置かれているから残るのではない。
紙に焼き付いているから残る。

この発想は、「紙を読む」という人間にとって最も強い基盤にデジタルを接続する試みである。

## What We Are Building

このプロジェクトは、短い音声や動画を紙に保存し、未来の端末上で復元できる仕組みをつくる試みである。

その中心には、次の構造がある。

- データ層: QRコードに埋め込まれたデジタルデータ
- 復元層: ブラウザ上のデコーダHTML
- 説明層: READMEやソースコードのような人間可読な仕様
- フォールバック層: パラパラ動画や紙面そのもの

つまりこれは、単なるQRユーティリティではなく、
**紙をメディアに変えるための設計**である。

## What Is Invented Here

このプロジェクトで本当に主張したい新規性は、単一の部品や単一のアルゴリズムではない。
価値があるのは、次の要素を一つの保存体系として結び付けた点にある。

1. 紙面上のQR群を、リンクではなくデータそのものの保存媒体として使うこと
2. 入口となる decoder HTML を公開し、サーバー常駐アプリに依存せずに復元できるようにすること
3. 復元処理を読み取った端末のローカルで完結させること
4. README やソースコードも紙へ残せるようにし、将来の再実装可能性まで保存対象に含めること

つまり発明の核は、
**「紙にデータを焼く」ことだけではなく、「紙・QR群・decoder・仕様書を合わせて自己修復可能な保存メディアにする」**
という全体設計にある。

単体では既存要素に見えるものでも、この組み合わせによって、

- クラウドが消えても残る
- 専用アプリが消えても再生系を作り直せる
- データだけでなく復元方法まで人間が読める形で残せる

という保存思想が成立する。

## Why It Matters

普通のデジタル記録は、サービス、サーバー、アプリ、フォーマット、アカウント、記録媒体に依存している。
どれか1つでも失われると、思い出へのアクセスは不安定になる。
デジタルは劣化しないと言われるがどうだろうか？
1990年代のホームページ、どれだけ残っているだろうか。
LD、CD、DVDなどの記録媒体、劣化してないだろうか。
紙なら1000年残せる可能性がある。

このプロジェクトは、その依存をできるだけ削る。

- サーバーに依存しない
- 専用アプリに依存しない
- 一企業の保存期間に依存しない
- ネット接続に依存しない方向へ寄せる

最終的には、「紙さえ残っていれば、未来の誰かが読み直せる」状態を目指す。

## Design Principle

判断に迷ったときは、次の原則を優先する。

- 残ることを、便利さより優先する
- 外部依存を、可能な限り減らす
- 人間が読めるものを残す
- 短い一瞬を、確実に保存できる方を選ぶ
- 画面上の見栄えより、最終的な読み取り成功率を優先する
- 総QR枚数だけでなく、1個あたりの物理サイズと密度も評価する
- 未来の再実装可能性を捨てない

## Long-Term Goal

### 機能面の長期目標

最終的な理想像はこうである。

1. 短い音声や動画を記録する
2. 動画はパラパラ動画として複数枚の紙面へ展開し、同時に各ページへQRを付ける
3. 音声や動画のデータ本体はQR群として紙面へ焼き付ける
4. decoder HTML を入口にして、何年後でも紙からデータを復元できる
5. README やソースコードも紙へ残し、仕組み自体をその時代の技術で再実装できる

### ユーザー体験としての長期目標

ユーザーから見た理想の流れはこうである。

1. 3秒ほどの音声や動画を撮る
2. その中の、ちょっとした仕草、揺らぎ、表情のような「ずっと残したい瞬間」を紙にする
3. 動画は複数枚のパラパラ動画へ展開され、各ページには対応するQRが印刷される
4. 紙を保管する
5. 紙をパラパラめくるだけでも、その記録の一部を見返せる
6. 何年後でも、音声や元動画を見たくなったら紙からデータへ復元できる
7. さらにソースコードや仕様も印刷してあれば、仕組み自体もその時代の技術で復元できる

これは、思い出をクラウドに置くのではなく、
**紙という最も素朴な媒体に、未来まで届くデジタルを宿す**ためのプロジェクトである。
