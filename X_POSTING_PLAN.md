# X Posting Plan

## 投稿の考え方

1. 1投稿目は「何が起きるか」を一番短く伝える
2. 1投稿目には、実際に試せる QR 画像と decoder URL を置く
3. 2投稿目で encoder と GitHub を案内する
4. 3投稿目が必要なら、印刷して復元できたことを補足する

## 添付画像のおすすめ

### 1投稿目

- `パラパラQR 音声チャンク (23 QR)_01.jpg`
- `パラパラQR 音声チャンク (23 QR)_02.jpg`
- 必要なら `03.jpg`

理由:

- 1ページ目に decoder 入口 QR が入っている
- 最初の数ページだけでも「紙に焼いてある」感じが伝わる

### 2投稿目

- エンコーダ画面のスクリーンショット
- デコーダ画面のスクリーンショット

### 3投稿目

- 印刷した紙の写真
- iPhone で復元している写真

## 文案

### 1投稿目

```text
あなたの、絶対に残したい1秒はなんですか？

短い音声を QR コード群として紙に焼き付けて、あとから別の端末で復元する試作を作りました。

ずっと、のこせるデジタル。
紙が、永遠のサーバーになる。

この画像の QR は実際に読めます。
まず decoder を開いてから、画像内の QR を順に読んでみてください。

decoder:
https://jtcpride.github.io/parapara-qr/parapara-qr-decoder.html
```

### 2投稿目

```text
自分でも試せるように、encoder と repo も公開しています。

encoder:
https://jtcpride.github.io/parapara-qr/parapara-qr-poc.html

repo:
https://github.com/jtcpride/parapara-qr

今できているのは、
- 短い音声を単一QR / 複数QRにする
- iPhone の decoder で読み取る
- ローカルに復元して保存する
- 紙に印刷したものから復元する

まだ改善中なのは、読み取りやすさ、録音ワークフロー、動画対応です。
でも「紙に保存して、あとから復元する」という核は、もう動いています。
```

### 3投稿目

```text
今回は「紙にデータを保存できるか」だけでなく、
印刷した紙から iPhone で実際に復元できるところまで確認しました。

クラウドに置くだけではなく、
現実世界の中で、未来まで届くデジタルを残せないか。

そのための試作です。
```

## 投稿前メモ

- 1投稿目に decoder URL を必ず入れる
- 1投稿目で encoder URL まで詰め込みすぎない
- 画像順は、decoder QR が入ったページを先頭にする
- 実際に復元できたノック音や短い `.m4a` を使う
