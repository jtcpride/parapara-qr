# パラパラQR PoC Phase 1

1. `parapara-qr-poc.html` をブラウザで開く。
2. 「📁 ファイル→QR生成」または「🎙 録音→QR」を押す。
3. 生成後「▶ テスト再生」で音声確認、必要なら「🖨 印刷」。

## Phase 2a の着手点

- `parapara-qr-decoder.html` を追加済み
- デコーダHTML上のボタンからカメラ起動を試せる
- `BarcodeDetector` が使える環境では QR 読み取り開始が可能
- iPhone Safari など `BarcodeDetector` 非対応環境では `jsQR` fallback で読取を試みる
- まずは手動で `data:text/html;base64,...` を貼り付けて復元する最小導線を用意
- 復元後は iframe プレビュー、音声/動画タグ、ダウンロード導線を表示
- `index.html` は decoder への入口ページで、GitHub Pages 公開後は `https://jtcpride.github.io/parapara-qr/` を入口QRにできる

公開中の入口URL:

- `https://jtcpride.github.io/parapara-qr/`

## 目指す実利用フロー

1. デコーダHTMLのURLを埋め込んだ入口QRを読む
2. iPhone 上でデコーダページを開く
3. デコーダ側のボタンからカメラを起動する
4. データ本体のQRを読む
5. 復元した音声・動画をその端末上で保存する

ブラウザの保存先は端末やブラウザ設定に依存するが、iPhone では通常 Files のダウンロード領域が候補になる。
入口QRを作るときは、上記の公開URLをそのまま埋め込めばよい。

## Phase 1 で検証できたこと

- 短い音声を `data:text/html;base64,...` 形式の自己完結QRとして1枚に埋め込める。
- QRの中には再生用HTMLと音声データ本体を同梱できる。
- 読み取ったペイロードを端末上で開けば、その端末上で音声を復元して再生できる。
- ブラウザ内の「テスト再生」と Playwright のラウンドトリップで、この往復を確認済み。

## まだ Phase 1 に含めていないこと

- 動画の自己完結QR化
- 複数QRへの分割と再結合
- iPhone標準カメラなど汎用カメラだけでの直接オープン

## Phase 2 の理想像

Phase 2 では、標準カメラに依存せず、専用のデコーダHTMLを入口にする。

想定フロー:

1. まず「デコーダHTML」を開く
2. 「QRを読んで元の音声・動画を復元する」ボタンを押す
3. そのHTML経由でカメラを開く
4. QRを読み取る
5. 端末ローカルでデータを復元して再生または保存する

理想的には、紙に印刷された入口QRを汎用カメラで読むとこのデコーダHTMLが開き、そこから復元ボタンで本体QR群を読む構成にする。
