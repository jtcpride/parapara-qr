# Phase 1 Milestone

## 固定したい到達点

Phase 1 では、単一の自己完結QRを使って「短い音声を紙や画面に焼き付け、別端末で復元する」という核を技術的に成立させた。

確認できたこと:

1. 単一HTMLのエンコーダで短い音声ファイル、または短時間録音から自己完結QRを生成できる
2. QRの中身は `data:text/html;base64,...` で、再生用HTMLと音声データ本体を同梱できる
3. GitHub Pages 上の decoder を入口として、iPhone Safari からデータQRを読める
4. 読み取った端末上で payload を復元し、元データをローカル保存できる
5. Playwright で Phase 1 の往復検証が通る

## Phase 1 の意味

これは「単一QRだけで完璧な実用品ができた」という意味ではない。

意味があるのは次の点:

- 紙や画面上のQRを保存媒体として使える
- データ本体をサーバーに置かなくても復元できる
- ブラウザだけで復元まで持っていける

つまり、プロジェクトの根にある
「紙が、永遠のサーバーになる」
という方向性は、少なくとも短い音声について成立した。

## 残った制約

- 単一QRは容量の壁がきつい
- 端末やブラウザ差で録音サイズがぶれやすい
- 復元後の `.webm` 再生互換性は iPhone で弱いことがある

## Phase 2 へ渡すもの

Phase 2 は、Phase 1 を捨てる段階ではなく、Phase 1 で確認できた核を実用側へ広げる段階として扱う。

次の主テーマ:

- 複数QR分割
- 複数チャンクの再結合
- Mac 先行での安定検証
- その後のモバイル読み取りUX改善
